# CialloClaw Prototype demo5

## 这是什么

一个“桌面天气机”原型，不是聊天窗口，不是悬浮球，而是一个持续存在但不打扰你的桌面协作状态表达器。

通过天气带的变化，让用户感知到“今天桌面气压怎么样”。

## 启动方式

### 一键启动（推荐）

双击项目根目录的 **run.exe** 即可：

```
E:\code\opencode\demo5\run.exe
```

它会自动：
1. 启动 Go 后端（监听 127.0.0.1:17888）
2. 调用 Qt 的 qml.exe 打开前端界面

### 手动启动

如果想分开启动：

1. 启动后端：
   ```cmd
   cd E:\code\opencode\demo5\backend
   E:\go\go1.24.13.windows-amd64\go\bin\go.exe run .
   ```

2. 用 Qt Creator 打开 `frontend/CMakeLists.txt`，或直接运行：
   ```cmd
   E:\Qt\6.8.2\mingw_64\bin\qml.exe frontend\qml\Main.qml
   ```

## 四个核心功能

### 1. 场景助手

根据当前场景（网页阅读、合同审阅、报错排查等）给出即时理解和建议。

**入口**：顶部天气带 + 今日态势图

### 2. 待办管家

把任务堆积情况做成“压力云层 / 临期风暴”的视觉表达。

**入口**：主界面下方的 TaskPressureView

### 3. 镜子记忆

告诉用户：“这是按你的习惯判断的”。

**入口**：主界面下方的 MemoryHintBlock

### 4. 安全卫士

高风险动作不做普通弹窗，而是进入“雷暴预演”，让用户看到影响范围和授权边界。

**入口**：点击风险操作后弹出的 StormPreviewOverlay

## 天气状态

| 状态 | 含义 |
|------|------|
| 晴   | 流畅，无需介入 |
| 多云 | 有帮助机会 |
| 阵雨 | 任务堆积、临期需推进 |
| 雷暴 | 风险动作、失败、异常 |
| 雾   | 上下文不完整，需澄清 |

---

## 三条演示链路

1. **日报整理**  
   进入场景 → 天气变阵雨 → 点击天气带展开态势图 → 点击“推进一步” → 展开日报草稿

2. **合同审阅**  
   进入场景 → 点击“生成确认稿” → 进入雷暴预演层 → 选择授权深度 → 确认

3. **报错排查**  
   进入场景 → 点击“自动修复建议” → 进入授权深度选择

## 技术说明

- **前端**：Qt6 + QML（纯 QML，不需要 CMake 编译）
- **后端**：Go 1.24（mock 数据）
- **通信**：本地 HTTP JSON（127.0.0.1:17888）
- **全部为 mock**：不接真实模型、OCR、系统权限
