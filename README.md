# CialloClaw Demo4 - UX 原型

基于 Wails 的桌面常驻 UX 交互原型。

## 启动方式

### 开发模式

```bash
# 进入项目目录
cd demo4

# 启动 Wails 开发模式
wails dev
```

### 构建生产版

```bash
# 构建前端
cd frontend
npm run build
cd ..

# 构建 Go 应用
go build .
```

## 功能说明

### 1. 悬浮球（默认常驻入口）

- **位置**：屏幕右下角
- **视觉**：泛光圆球 + 简约表情 + 两根线条耳朵
- **动效**：呼吸光、眨眼、耳朵轻微摆动

#### 交互规则

| 操作 | 效果 |
|------|------|
| 鼠标靠近 | 出现轻量对话气泡（不是大窗口） |
| 双击 | 打开工作台（窗口1） |
| 右键 | 显示菜单：隐藏 / 退出 |
| 左键（感叹号态） | 打开承接确认窗（窗口2） |
| 拖入材料 | 变成加号，点击后打开窗口2 |

### 2. 轻量对话气泡（Hover 触发）

- 出现在悬浮球旁边
- 半透明玻璃感
- 内容：最近一轮的用户输入 + Agent 回复
- 交互：输入框 + Enter 发送 + 关闭按钮

### 3. 窗口2：承接确认窗

触发条件：感叹号态单击 / 拖入材料

结构：
- **A区**：输入材料卡片（可删除单项）
- **B区**：意图判断 + 修改输入框
- **C区**：圆形确认按钮（✓）

### 4. 窗口3：小型回复窗

触发条件：窗口2 点击确认

- 显示 agent 的轻量回复
- 右下角「打开工作台」按钮
- 「加入待办」按钮

### 5. 工作台（窗口1）

触发条件：双击悬浮球 / 窗口3 点击打开工作台

#### 顶部下拉菜单
- 会话（默认）
- 状态
- 待办
- 设置

#### 会话模式
- 历史会话列表
- 当前对话区
- 输入框 + 快捷意图按钮

#### 状态模式
- 空闲 / 工作中 / 故障
- 工作中显示具体任务
- 故障显示失败说明

#### 待办模式
- 手动新增
- 今日 / 稍后 / 已完成
- agent 建议标记

#### 设置模式
- 悬浮球显示开关
- 提醒开关
- 记忆能力开关
- 版本信息

### 6. 开发调试入口

位置：左上角「DEV」按钮

功能：
- 触发感叹号
- 触发加号
- 默认态
- 打开窗口2
- 打开窗口3
- 打开轻量对话
- 打开工作台

## 技术栈

- **Go**：Wails v2 桌面应用宿主
- **前端**：HTML + CSS + 原生 JavaScript
- **无框架**：不使用 React/Vue/Tailwind/Bootstrap
- **静态数据**：mock 数据，无后端接口

## 项目结构

```
demo4/
├── main.go           # Wails 入口
├── app.go           # 应用逻辑

├── go.mod / go.sum  # Go 依赖
└── frontend/
    ├── index.html
    ├── style.css
    ├── app.js       # 主逻辑
    ├── state.js     # 状态管理
    ├── data.js      # Mock 数据
    ├── utils.js     # 工具函数
    ├── views/
    │   ├── desktop.js      # 悬浮球
    │   ├── quickChat.js   # 轻量对话

    │   ├── resultPopup.js  # 窗口3
    │   ├── workspace.js   # 工作台
    │   ├── devPanel.js    # 调试入口
    │   └── panels/        # 工作台各模式
    └── dist/              # 构建产物
```

## 交互闭环

```
启动 → 仅显示悬浮球
  │
  ├─→ 鼠标靠近 → 轻量对话气泡
  │       └─→ 输入/发送/关闭
  │
  ├─→ 双击 → 工作台（会话模式）
  │       └─→ 菜单切换/关闭
  │
  ├─→ 调试触发感叹号 → 单击 → 窗口2
  │       └─→ 确认 → 窗口3 → 工作台
  │
  └─→ 拖入材料 → 窗口2
          └─→ 确认 → 窗口3 → 工作台
```

## 注意事项

1. 程序启动后默认只显示悬浮球，不覆盖全屏
2. 所有小窗必须明确可关闭
3. 记忆不是高频主舞台，是长期协作能力
4. 高风险执行需要确认感，但本次是静态 mock
