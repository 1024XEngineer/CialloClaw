# CialloClaw 用户指南

这份指南只覆盖最基础的启动、使用和常见问题。更完整的架构、协议和开发约束仍以 `docs/` 下的核心设计文档为准。

## CLI 启动

首次拉取仓库后，在仓库根目录安装依赖：

```bash
corepack enable
pnpm install
```

启动官网预览：

```bash
pnpm dev:website
```

启动桌面端开发入口：

```bash
pnpm dev:desktop
```

单独启动本地服务：

```bash
pnpm dev:service
```

如果需要同时验证桌面端和本地服务，建议开两个终端：一个运行 `pnpm dev:service`，另一个运行 `pnpm dev:desktop`。

## 基本使用

1. 打开桌面端后，先从悬浮球开始交互。
2. 可以通过文本输入、语音表达、文件拖拽或选中文本把现场上下文交给 CialloClaw。
3. 当输入需要执行、确认、授权或持续追踪时，系统会围绕 `task` 创建或更新正式任务。
4. 高风险动作会进入授权和审计流程，不会静默执行。
5. 执行结果会以短结果、`delivery_result` 或 artifact 的形式进入气泡、仪表盘或任务详情。

## 常见问题

### 我只想看官网，不启动桌面端可以吗？

可以。运行 `pnpm dev:website` 后访问 Vite 输出的本地地址即可。

### 为什么有时需要先启动 local-service？

桌面端的正式任务、运行编排、风险治理和交付结果都由 Go local-service 承接。只看静态界面时可以不启动；验证真实主链路时需要启动。

### 文件拖拽会不会立刻执行？

不会。文件拖拽会先进入附件队列，用户手动发送并完成必要确认后，才会进入任务创建和后续执行链路。

### 结果在哪里看？

短结果优先在气泡里看；正式交付、artifact、审计摘要和任务进度可以在仪表盘或任务详情里查看。

### CLI 命令失败怎么办？

先确认 Node、pnpm 和 Go 已安装，再重新运行 `pnpm install`。如果是桌面端或服务端链路问题，优先查看终端输出的端口、Named Pipe、权限或依赖错误。
