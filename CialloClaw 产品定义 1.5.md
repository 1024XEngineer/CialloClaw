# 1. 背景

当前最容易使用 Agent 的人群主要有两类：一类是**信息和文档密集型岗位**，例如咨询、研究、内容、产品；另一类是**流程和交付密集型岗位**，例如运营、销售支持、客服、行政和数据运营。

## 1.1 核心目标用户定义

本产品的核心目标用户，是一类**以任务完成和效率提升为导向的用户群体**。他们关注的重点不是 AI 的技术概念本身，也不是模型能力的上限展示，而是 AI 是否能够在真实场景中**稳定、低门槛、低风险地帮助自己完成任务**。

这类用户通常因“AI 能替我做事”而产生兴趣，但决定是否持续使用的关键，在于产品是否真正具备可落地的执行价值，包括是否足够易用、是否安全可控、是否成本透明，以及是否能够持续嵌入日常工作流。

从本质上看，他们不是为“聊天”而来，而是为“省事”而来；不是追求炫技体验，而是追求明确、持续的效率收益。



虽然产品面向的是同一类“任务驱动的效率用户”，但在使用深度上可进一步划分为两层：

* 轻量效率用户

这类用户的核心诉求是“尽快省事”。他们有明确任务，但不愿投入过多时间学习复杂功能，也不希望经历繁琐配置。他们更偏好**简单、直接、可立即见效**的产品体验，可通过**主动协助、快捷任务入口、默认模板能力**等方式满足需求。

* 进阶效率用户

这类用户不仅希望 AI 帮助完成单点任务，还希望将 AI 纳入自己的长期工作流。他们愿意配置技能和插件，希望产品支持更灵活的任务编排和跨应用协作。这部分用户是产品形成长期粘性和使用深度的重要人群。

## **1.2 用户痛点**

* **上手门槛高**

  许多现有的桌面 Agent 产品在使用前需要用户进行复杂的安装、配置和学习较多的操作步骤和概念。例如，需要配置工作区、连接各类服务、理解插件机制等。对于普通用户来说，这些步骤过于繁琐，造成了极高的**入门门槛**，导致用户难以快速上手并投入实际使用，尤其是对于缺乏技术背景的用户。&#x20;

* **使用习惯门槛高**

  即便产品能够被成功安装并配置完成，用户仍然面临着**使用习惯的障碍**。很多用户并不习惯在日常工作或生活中主动调用 Agent 来协助解决问题。用户倾向于依赖传统的操作方式，例如直接搜索、手动处理信息、拖放文件等，难以将 Agent 集成到他们的日常流程中。这就导致了用户在需要时可能会忘记使用 Agent，从而影响产品的使用频率和效果。&#x20;

* **触发不合理**

  目前大多数 Agent 产品的交互触发方式存在问题。要么过度依赖用户主动提问和发起任务，使得 Agent 处于“被动等待”状态，难以在用户需要时及时出现；要么当 Agent 主动介入时，常常在不恰当的时机出现，导致打扰用户的正常操作。例如，某些产品会在用户正在处理复杂任务时突然弹出对话框或通知，反而增加了用户的负担，而没有解决实际问题。这种不合理的触发方式，往往让用户感到烦躁，降低了产品的可用性和满意度。&#x20;

* **重复性操作和上下文切换成本高**

  在日常工作中，用户通常会遇到大量重复性操作，例如复制粘贴信息、整理文件、在不同网页和应用之间来回切换、反复确认下一步该做什么等。每当需要从一个工具切换到另一个工具，或在不同任务之间往返时，都需要重新整理上下文、回忆操作步骤并衔接当前进度，这会持续消耗时间和注意力。用户希望在当前任务现场获得更及时的提示、内容整理、下一步建议和相关资料快速打开能力，从而减少机械性重复劳动和频繁切换带来的负担。

* **执行可控性与安全感不足**

  随着 Agent 参与更多实际执行，用户会进一步关注文件落点是否清晰、危险操作是否隔离、不同风险动作是否有明确确认机制，以及执行失败后是否能够恢复。若缺少清晰的写入边界、权限分级和回滚兜底，用户通常只愿意把 Agent 当作辅助工具，而不会放心将关键任务交由其协助完成。

## 1.3 解决方案

1. **轻量化设计：降低安装、配置和理解成本，让普通用户快速上手**

   我们的产品通过简化安装和配置过程，最大限度地减少用户在启动和使用过程中的复杂步骤。与传统的桌面 Agent 产品相比，CialloClaw **无需复杂的权限设置、插件安装或工作区配置**，普通用户只需简单几步就可以完成产品的初始化和配置，立刻进入可用状态。我们还设计了直观的用户界面和操作流程，用户可以在没有技术背景的情况下，轻松理解并开始使用。这样，产品不仅降低了**上手门槛**，还让用户能迅速体验到其核心功能，避免因繁琐操作而产生的**学习成本**。&#x20;

2. **低负担交互：减少用户把需求说清楚并交给 Agent 执行的成本**

   传统的桌面 Agent 产品往往要求用户明确地表达需求，并通过复杂的交互操作将任务传递给 Agent。CialloClaw 通过 **上下文感知** 和 **自然语言处理技术**，让用户不再需要明确指定每个操作步骤或任务要求。系统会主动捕捉用户当前的任务场景（如正在处理的文件、网页或邮件），并根据上下文自动推测用户的需求，主动提出帮助，而不是让用户反复描述需求。通过这种方式，用户仅需点击几下，即可将任务交给 Agent 处理，**大大减少了需求表达的成本**，提升了操作效率。&#x20;

3. **低打扰主动协助：基于上下文在合适时机提供帮助，而不是一味等待提问或频繁打扰**

   现有的 Agent 产品往往在用户遇到问题时被动等待，而在主动介入时又容易打扰用户。CialloClaw 打破这一局限，采用 **低打扰的主动协助机制**。系统会根据用户当前的上下文（例如正在浏览的页面、编辑的文件或正在进行的任务）主动感知帮助机会，并在合适时机给出提示或建议，而不是频繁弹窗或强行打断用户。**这种设计让用户在无需主动提问的情况下，能够在真正需要帮助时得到及时的支持**，避免了用户的反感，也提升了产品的实用性和满意度。&#x20;

4. **可控执行与分级安全：降低关键任务交付顾虑**

   针对 Agent 参与执行后带来的可控性和安全顾虑，CialloClaw 通过清晰工作区边界、执行面沙盒化、风险分级授权以及快照回滚机制，降低文件误写、环境污染和高风险误操作带来的使用阻力。在此基础上，产品既保持桌面常驻和低打扰体验，又能让 Agent 在更可控的前提下参与实际任务推进。

# 2. 市场调研与竞品分析

## 2.1 市场现状：当前 Agent 产品的主流解法

近几年，Agent 产品的重点已从单纯对话转向任务执行、流程编排与跨应用协作。当前市场主流解法主要包括：

* **计算机使用型 Agent**
  &#x20;以浏览器或桌面 GUI 为操作对象，通过界面理解与鼠标键盘控制完成多步任务，强调自动执行与流程外包。&#x20;

* **工作区 / 平台式 Agent**
  &#x20;以工作区、任务面板、技能系统、定时任务和 workflow 编排为核心，强调“规划—执行—交付”的任务闭环。&#x20;

* **轻量引擎 / 多入口接入型 Agent**
  &#x20;强调低资源占用、跨平台部署和多入口接入，通常通过 CLI、WebUI 或消息入口接收任务，更接近 Agent 引擎能力而非桌面原生协作产品。

整体来看，市场已经验证了执行型 Agent 的需求成立，但不同路线的重点主要集中在执行能力、平台组织能力和引擎接入能力，产品形态分化明显。

## 2.2 市场空缺与机会判断

尽管当前 Agent 市场已经具备任务执行、流程编排和跨应用协作能力，但市场仍缺少一种能够以**低门槛、低打扰、低切换成本**进入真实任务现场，并在长期使用中形成稳定协作关系的产品形态。与此同时，随着 Agent 逐步从“提供建议”走向“参与执行”，市场对于执行边界清晰、风险分级明确、结果可恢复的产品能力也提出了更高要求。&#x20;

当前市场的主要机会体现在以下几个方面：

* **缺少桌面现场承接型 Agent**
  &#x20;现有产品大量依赖消息入口、工作区、浏览器侧边栏或专门平台承接任务，而不是直接在正在发生的桌面任务环境中自然介入。围绕桌面现场承接协作的产品形态仍然相对不足。

* **缺少低打扰主动协助的统一体验**
  &#x20;市场需要的不是更强烈的打断式提醒，而是在合适时机、以合适强度提出建议，再由后续确认承接执行。现有市场虽已有主动协作探索，但“轻提示、低打扰、确认后执行”的统一体验仍未成为主流。

* **缺少从理解、提醒到轻执行的连续链路**
  &#x20;很多产品要么停留在信息辅助层，要么直接强调复杂自动化执行，中间缺少一条从上下文理解、任务提醒、下一步建议到轻执行支持的连续产品路径。

* **缺少以长期协作为目标的 Agent 形态**
  &#x20;主流产品普遍更强调单次任务闭环或执行能力，而长期习惯学习、流程沉淀和“越用越顺手”的协作关系仍未成为统一的主流方向。

* **缺少兼顾轻量用户与进阶用户的成长路径**
  &#x20;一部分产品偏重平台和配置，更适合进阶用户；另一部分产品偏重引擎能力和接入效率，更适合技术用户。市场仍存在一种既能让轻量用户快速获得帮助，又能让进阶用户逐步沉淀流程和协作习惯的产品机会。

* **缺少可控执行的桌面 Agent 体验**
  &#x20;随着 Agent 参与更多实际操作，市场不仅需要“能执行”的产品，也需要“执行边界清晰”的产品。用户会越来越关注文件落点是否明确、危险操作是否与宿主环境隔离、不同风险动作是否有分级确认，以及执行失败后是否具备恢复能力。当前补充的工作区边界、执行面沙盒化、策略引擎和快照回滚能力，正说明这一方向已经成为产品竞争力的重要组成部分。

概括而言，当前市场并不缺少“能做事”的 Agent，真正缺少的是一种能够**在桌面环境中自然出现、持续协作，并在可控前提下参与执行**的产品形态。这构成了后续关键竞品对比和产品切入点判断的基础。&#x20;

## 2.3 关键竞品对比

**相对 OpenClaw**，CialloClaw 的价值不在底层能力本身，而在于把底层框架产品化为可被持续使用的桌面 Agent 形态；前者偏能力底座，后者偏前台协作体验。

**相对 PicoClaw**，CialloClaw 的差异不在部署轻量性，而在于桌面原生入口、任务现场承接和低打扰主动协助。

**相对 QoderWork / DuMate**，CialloClaw 不以重工作区为前提，而是以桌面常驻形态直接嵌入原有工作流。

**相对 Claude Cowork**，CialloClaw 更强调个人长期协作、习惯学习和低存在感陪伴，而不是平台化任务管理。&#x20;

## 2.4 CialloClaw 的切入点与独特价值

基于前述市场空缺与竞品对比，CialloClaw 的切入点并不在于继续叠加更多工具调用能力，而在于重构 Agent 进入任务流的方式：**以桌面常驻作为主入口，以低打扰主动协助作为主交互，以长期习惯学习作为主增益机制，形成区别于引擎型、工作区型和平台型 Agent&#x20;**&#x7684;产品路径。&#x20;

CialloClaw 的独特价值主要体现在以下几个方面：

* **桌面原生的协作入口**
  &#x20;产品以常驻桌面的悬浮球、对话悬浮窗和气泡提示作为核心 UI 形态，不要求用户长期停留在独立平台或工作区中，而是在原有桌面环境中承接协作。

* **低打扰的主动协助机制**
  &#x20;产品通过持续感知当前页面、文件、选中内容、复制行为、任务状态和系统状态，识别帮助机会后先以轻提示提出协助，再由后续确认承接执行，核心逻辑是“先提示，再确认，后执行”。&#x20;

* **围绕真实任务现场提供帮助**
  &#x20;产品并非等待任务被完整描述后再执行，而是围绕当前任务环境直接提供提醒、总结、解释、下一步建议、资料打开和轻量执行支持，减少需求表达成本与上下文切换成本。&#x20;

* **从单次任务走向长期协作**
  &#x20;产品通过“另世我 / 镜子”机制，对任务路径、操作习惯和处理方式进行持续沉淀，在后续相似场景中提供更贴近个人方式的建议、提示或半自动工作流，使 Agent 从单次响应逐步过渡到长期协助。&#x20;

* **同时覆盖轻量使用与进阶扩展两类需求**
  &#x20;一方面，产品通过轻量化入口、低负担交互和低存在感常驻降低首次使用门槛；另一方面，又通过任务巡检、流程沉淀和后续可扩展的工作流能力支撑更深度的长期使用。&#x20;

综合来看，CialloClaw 的差异化不在于单项能力的绝对领先，而在于以**桌面常驻 + 主动协助 + 长期协作**的统一形态，降低 Agent 进入真实任务流的成本，并提升其持续使用的可能性。相较于 PicoClaw 的引擎化路径、QoderWork / DuMate 的工作区路径以及 Claude Cowork 的平台化协作路径，CialloClaw 更接近“桌面任务现场中的长期协作 Agent”。&#x20;

# 3. 产品定义

## 3.1 产品形态与交互入口

CialloClaw 是一款**以桌面常驻为入口，以任务现场承接为方式，以低打扰协作为体验，以长期陪伴和可控执行为方向的桌面 Agent 产品**。

它不是一个要求用户长期停留其中操作的独立网页平台，也不是以命令行、聊天窗口或重工作区为主要入口的工具，而是一个直接嵌入用户原有桌面工作流中的协作型助手。

从用户视角看，CialloClaw 主要由四个可感知的产品部件组成：

* **悬浮球**

  * 三类作用

    * &#x20;通过角色表情、状态微指示器和细微动画表达当前状态；&#x20;

    * &#x20;作为用户主动进入产品的第一入口；&#x20;

    * &#x20;作为任务环、轻提示和对话弹出层的锚点。

  * 作为默认常驻入口，停留在桌面边缘，以低存在感方式持续显示当前状态，并支持点击、悬停、拖拽和右键操作。&#x20;

* **任务环**
  &#x20;当用户悬停或触发悬浮球时，周围展开常用任务入口，用于快速发起总结、翻译、提问、提醒等高频操作。
  &#x20;它的价值不在于展示全部能力，而在于把最常用、最短链路的操作放在离用户最近的位置，降低发起任务的成本。

* **轻提示卡片**
  &#x20;当系统识别到明确帮助机会时，不直接打断用户，而是以非模态、低侵入的方式给出提示，用户可选择忽略、稍后处理或点击进入进一步操作。
  &#x20;它是产品主动协助的主要外显方式，用来在“有帮助价值”和“不过度打扰”之间取得平衡。

* **对话弹出层**
  &#x20;用于承接具体任务、展示结果、确认执行和进行多轮交互，是产品的主要任务处理界面。
  &#x20;它不是一个独立工作区，而是围绕当前桌面环境展开的轻量任务面板，用来承接更深入的协作。

整体来看，CialloClaw 的产品形态不是把功能堆在一个大平台里，而是通过悬浮球、任务环、轻提示卡片和对话弹出层之间的分工，形成一套由浅入深、从低打扰到深度承接的桌面交互结构。

## 3.2 核心功能

CialloClaw 的核心功能围绕“**陪伴 + 助手**”展开，不以单一对话能力为中心，而是通过场景理解、任务推进、长期记忆与可控执行，形成一套面向桌面任务环境的连续协作能力。结合当前产品定义，核心功能可概括为以下四项：

### 3.2.1 场景助手

场景助手负责围绕当前任务环境提供即时帮助，是产品最直接面向高频桌面任务的核心功能。它的重点不在于让用户先补充完整上下文，再交由系统处理，而是在页面、选中文本、复制内容、上传文件、程序状态、错误界面或停留行为中识别明确线索，并在合适时机给出总结、解释、翻译、提炼与下一步建议。它既服务内容理解，也服务问题排查与任务推进。

在**能力**上，场景助手主要覆盖四类输出：

* 对网页、文档、代码片段和选中文本进行总结、翻译、解释与重点提炼；

* 对错误信息、终端报错和异常界面提供原因分析与处理建议；

* 从当前内容中提取关键信息、待办和下一步动作；

* 在系统资源异常、网络波动等情况下给出状态提醒与处理建议。换句话说，它不只是“告诉用户发生了什么”，而是尽量把帮助推进到“现在可以做什么”这一层。

**交互方式**上，场景助手遵循“低侵入、先提示、后展开”的原则：

* &#x20;通过悬浮球状态变化表达当前场景中存在帮助机会；&#x20;

* &#x20;通过轻提示卡片给出简短建议或待确认动作；&#x20;

* &#x20;通过任务环承接总结、翻译、提问等高频短链路操作；&#x20;

* &#x20;在用户确认后进入对话弹出层，进行更深入的解释、草稿生成或任务承接。
  &#x20;同时，系统优先基于选中、复制、上传、报错、截止临近等强意图信号触发，而不是频繁打断用户。涉及外部操作时，也必须进入确认流。

### 3.2.2 待办管家

待办管家负责承接任务文件、待办清单和周期性巡检，是产品从“提醒”走向“推进任务”的核心功能。它的重点不在于单纯记录待办，而在于帮助识别优先级、提示临近事项，并围绕当前任务提供下一步协助，使任务从“被记下”进一步转向“被推进”。它补的不是“提醒”本身，而是从任务记录到任务行动之间的那一段空白。

在**能力**上，待办管家主要包括：

* 定时读取和解析指定的 `.md` 任务文件；

* 识别任务项、日期、状态、优先级和标签；

* 生成今日任务摘要、临期提醒和优先级建议；

* 在需要时打开相关文件、网页、目录，或生成消息草稿、邮件草稿、会议前清单等承接结果。

**交互方式**上，待办管家遵循“持续巡检、低打扰提醒、按需展开承接”的原则：

* &#x20;通过悬浮球状态变化或轻提示卡片提示任务临近、任务未完成或今日摘要；&#x20;

* &#x20;通过任务环或快捷入口快速进入任务相关操作；&#x20;

* &#x20;在用户确认后进入对话弹出层，查看更完整的任务摘要、优先级建议和下一步动作；&#x20;

* &#x20;在需要时直接打开相关资料、网页、目录，或生成草稿内容。
  &#x20;它优先围绕“今天到期”“即将超时”“长时间未处理”等明确时间线索触发，不默认接管全部任务，也不默认进行高风险自动执行。

### 3.2.3 镜子记忆

镜子记忆是产品面向长期协作的核心能力。它的目标不是保存聊天记录本身，而是通过结构化沉淀、阶段性总结和智能召回，逐步形成对用户偏好、任务状态和工作方式的持续理解，从而支持个性化理解、连续任务承接、习惯学习和长期陪伴。

在能力上，镜子记忆主要包括三层：**短期记忆、长期记忆和镜子总结**。

短期记忆用于连续理解当前任务，长期记忆用于沉淀用户偏好、稳定能力特征和反复出现的工作模式，镜子总结则用于周期性输出阶段总结和用户画像。

其核心机制可以概括为：**信息提取 → 分类存储 → 记忆压缩 → 检索召回**。这样既能让系统在相似任务中更懂用户，也能避免无意义历史不断堆积。

**交互方式**上，镜子记忆默认在后台运行，不以高频显式交互为主：

* &#x20;在日常使用中自动提取并沉淀可复用信息；&#x20;

* &#x20;在相似任务再次出现时，通过建议、提示和任务承接体现记忆效果；&#x20;

* &#x20;在周期性总结场景中，以总结报告或偏好概览让用户感知长期价值；&#x20;

* &#x20;在控制面板中提供记忆查看、记忆管理和偏好设置入口。
  &#x20;同时，记忆不是无限存储，用户应能够查看、修改、删除或关闭记忆能力，系统也需要控制召回范围与相关性，避免记忆污染当前任务。

### 3.2.4 安全执行模式

安全执行模式负责为 Agent 的实际操作建立基本边界和兜底能力，使产品在参与执行时不仅“能做”，也“可控”。它不是把产品做成一个重型安全平台，而是在桌面协作基础上，为文件写入、命令执行、网页自动化和系统动作提供必要的风险分级、执行隔离与恢复机制。

在**能力**上，安全执行模式主要包括四个方面：

* 将用户成果文件与应用运行数据分层存储，并通过工作区管理保证文件落点清晰可见；

* 通过执行面沙盒化隔离高风险动作，避免直接污染宿主环境；

* 通过红黄绿灯授权系统对不同风险等级的操作进行静默执行、授权确认或强制拦截；

* 在高风险动作前创建快照，在失败或异常时支持恢复与回滚。

**交互方式**上，安全执行模式不单独追求存在感，而是嵌入其他能力的执行链路中：

* &#x20;低风险动作可在明确规则下静默执行；&#x20;

* &#x20;中高风险动作必须通过轻提示或对话弹出层进入确认流；&#x20;

* &#x20;用户在确认界面中应能理解风险等级、目标对象和执行后果；&#x20;

* &#x20;执行后可在控制面板中查看日志、状态与恢复入口。
  &#x20;它的核心不是让 Agent 更激进地自动化，而是让用户在关键任务中敢于交出部分执行权，因为系统在边界、确认和恢复上都是清晰的。

总体而言，CialloClaw 的核心功能并不是若干离散工具的简单堆叠，而是围绕“理解当前场景、推进当前任务、积累长期习惯、保障执行边界”构建的一套桌面协作能力。这些功能共同定义了产品的核心使用价值：在不脱离当前工作环境的前提下，帮助用户更自然地完成日常任务，并在长期使用中形成持续协作关系。

## 3.3 核心任务场景

CialloClaw 的核心任务场景并不围绕单一对话展开，而是围绕用户已经处于工作流中的桌面环境展开。产品最适合在用户**正在阅读、正在处理任务、正在排查问题、正在切换应用或正在等待结果**的过程中介入，通过低打扰方式提供理解、提醒、建议和轻量承接。整体上，核心任务场景可归纳为以下四类。

### 3.3.1 内容理解与即时处理

当用户处于阅读网页、浏览文档、查看技术资料、选中文本或复制片段内容的场景时，CialloClaw 提供即时的内容理解与处理支持。该场景的核心价值在于减少重复性的上下文补充和内容搬运，让总结、翻译、解释、提炼和草稿生成能够在当前页面或当前应用附近直接完成，而不必先离开现场进入独立工具。&#x20;

这一场景主要覆盖：

* &#x20;网页、文档、代码片段的总结、改写、翻译与解释；&#x20;

* &#x20;从选中文本中提取关键信息和结构化要点；&#x20;

* &#x20;围绕当前内容生成回复草稿、会议清单或下一步建议。

### 3.3.2 任务巡检与任务推进

当用户通过 Markdown 任务文件、待办清单或阶段性工作安排管理事务时，CialloClaw 适合作为持续巡检与任务推进工具使用。其核心不只是“提醒还有哪些任务”，而是围绕任务状态给出更可行动的承接，例如生成摘要、识别临期事项、打开相关资料、提供下一步建议，并帮助任务从记录状态走向推进状态。

这一场景主要覆盖：

* &#x20;开工前、会前、截止临近时的任务简报与提醒；&#x20;

* &#x20;对任务文件的定时读取、结构识别和状态判断；&#x20;

* &#x20;围绕当前任务打开文件、网页、目录或生成草稿内容。

### 3.3.3 异常发现与问题排查

当用户遇到错误弹窗、终端报错、页面异常、长时间停留或明显重复失败时，CialloClaw 适合作为问题排查与下一步协助工具使用。该场景的重点不是自动接管故障处理，而是在错误发生或卡住时及时提供解释、原因分析、排查建议和上下文整理，降低用户在问题定位过程中的中断成本和切换成本。&#x20;

这一场景主要覆盖：

* &#x20;对报错信息、异常界面和失败结果进行解释与初步分析；&#x20;

* &#x20;围绕当前问题整理上下文、归纳尝试记录并给出下一步建议；&#x20;

* &#x20;在涉及系统动作或高风险操作时，仅提供建议或进入确认流，不直接默认执行。

### 3.3.4 长期习惯学习与持续协作

当产品在用户的长期使用中持续积累任务线索、交互记录和处理方式后，CialloClaw 适合作为个性化协作助手使用。该场景的重点不在于单次任务完成，而在于通过周期性总结、习惯提炼和相似场景召回，使产品逐步理解用户常见的任务路径、工作方式和决策顺序，从而形成“越用越顺手”的长期协作关系。

这一场景主要覆盖：

* &#x20;周期性生成总结报告和用户画像；&#x20;

* &#x20;从重复行为中归纳高频路径、工作模式和偏好；&#x20;

* &#x20;在相似任务再次出现时，基于历史习惯提供更贴近个人方式的建议与准备支持。&#x20;

总体而言，CialloClaw 的核心任务场景并不是“代替用户完成所有操作”，而是**在桌面工作流中承接理解、推进、提醒和协作**。其最核心的产品价值，在于让帮助发生在任务现场、发生在合适时机，并在长期使用中逐步演变为稳定的桌面协作关系。&#x20;

## 3.4 典型用户故事

### 故事 1：我想要一个装好就能用、顺手就能帮上忙的工具

我是一名电商运营，平时工作很杂，每天都要在聊天软件、表格、文档、网页后台和文件夹之间来回切。

&#x20;我不怕用新工具，但不想先花时间学习设置和规则。我更希望它下载安装简单，打开就能用，我直接说一句“帮我整理一下”“帮我找个文件”“帮我顺一下这段话”，它就能马上开始帮我。

对我来说，最有用的不是功能有多复杂，而是能不能顺手帮我把眼前的事往前推一点。

&#x20;比如我在看文档时，它能帮我总结重点；我在整理资料时，它能顺手归类；我要写东西时，它能先帮我打个草稿。这样我就不用为了一个小动作反复切来切去。

另外，很多事情平时散在聊天记录、表格、文档和文件夹里，看着都在推进，其实很容易漏。

&#x20;所以如果它还能顺手帮我看一眼，提醒我哪些事情快到时间了、哪些内容还没补齐、哪些任务卡住了，我会轻松很多。

所以我最容易有的感觉是：
**“它装好就能用，不用我先学；我一打开，它就能顺手帮我总结文档、归类资料、打草稿，还能提醒我哪些事情快漏了。”**

### 故事 2：我希望它越来越懂我，但别太烦，也别乱动我的东西

我是一名行政管理人员，平时要处理很多琐碎但不能出错的事情，比如整理文件、跟进流程、确认事项和同步信息。

&#x20;我希望这类工具不是只帮我一次，而是能在长期使用中慢慢了解我的做事习惯。这样下次碰到类似的事情，就不用我每次都重新讲一遍，它能更自然地接上，真正变成一个能长期帮我分担事情的助手。

但对我来说，分寸感也很重要。

&#x20;我不希望它频繁弹出来打扰我，也不希望它没经过我同意，就自己去改文件、动内容，或者直接处理一些关键操作。更理想的状态是，它该提醒我的时候提醒一下，也让我知道为什么提醒；我不需要的时候，也可以把提醒关掉，或者少一点。

碰到重要的事情，它最好先问我一句，再继续往下做；做完之后，也能告诉我它做了什么、结果是什么。

&#x20;如果真的不小心删错了东西，或者哪里处理错了，也最好还能恢复回来，这样我用起来才会更安心。

所以对我来说，真正好的感觉是：
**“它会慢慢懂我，能帮我做事，但不会一直打扰我，也不会在关键地方自己乱来；就算出了问题，也还能恢复回来。”**

## 3.5 产品边界

CialloClaw 的定位，不是命令行驱动的 Agent 引擎，也不是以工作区、任务面板和复杂编排为中心的重型协作平台。产品强调在原有桌面环境中自然承接协作，而不是要求用户先进入一个独立系统，再将任务显式交付给 Agent。

同时，CialloClaw 也不以高侵入、强接管式的自动执行为核心目标。其重点在于通过轻提示、任务建议、内容处理、任务推进和有限度的执行支持，帮助用户更顺畅地完成任务；对于高风险操作，则通过分级授权、执行隔离和恢复机制建立基本边界。

因此，CialloClaw 的产品边界可以概括为：**以桌面常驻为入口，以低打扰协作为主要交互方式，以任务现场承接和长期协作为核心目标，并在可控前提下参与有限执行的桌面 Agent 产品。**
