# CialloClaw 架构文档包

本目录包含 10 份核心 Markdown 文档：

1. `00-总览与开发路线.md`
   - 项目定位
   - 分层原则
   - 总体架构
   - 开发路线
   - 功能扩展总策略

2. `01-核心状态模型与状态机.md`

   - 状态机
   - 关联键设计

3. `02-事件总线与内部接口协议.md`

   - Payload 设计
   - 协议与版本化建议

4. `03-扩展开发指南.md`

   - 示例代码
   - 插件式注册方式

5. `04-事件字典.md`
   - 统一事件命名与事件头

   - 允许的直接调用边界
   - 新功能应落在哪一层、如何接入

7. `06-Repository与持久化设计.md`
   - 本地优先的持久化方案
   - SQLite / 文件系统 / 可选向量存储划分
   - Repository 接口设计
   - 事务、索引、恢复与清理策略

8. `07-启动与运行时装配.md`
   - main.go 与 bootstrap 分工
   - 启动顺序、注册顺序、恢复顺序
   - RuntimeComponent / Recoverable 抽象
   - 关闭流程与扩展接入点

9. `08-扩展接口规范.md`
   - 扩展类型、元数据与生命周期
   - Tool / Agent / Perception / Middleware / Recovery 接口约束
   - 注册、配置、事件、状态与权限边界
   - 新增功能的标准接入方式

10. `09-错误处理与恢复策略.md`
   - 统一错误分类与严重性分级
   - 各层失败处理、重试、补偿与熔断策略
   - Session / Task / Loop / Approval 状态映射
   - 启动恢复顺序与对象级恢复规则
